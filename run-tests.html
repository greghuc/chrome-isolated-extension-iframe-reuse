Test runner

Important note

For this test extension to run properly, you need a 'static' web-server running, to represent a 'remote' website. To do this, run

  • cd location-of-this-extension-on-disc
  • ruby -run -ehttpd . -p8000
We're using ruby to run a static web-server on port 8000, returning files from the extension folder. If you don't have ruby, try another one-liner.

What's going on

We're testing interaction between an extension background-script and a content-script running inside a webpage. The webpage is loaded inside an iframe on the extension's background page. On webpage load, the content-script runs, opens a port, and sends a message back to the extension's background (value of document.body.outerHTML).

We expect the extension background-script to receive the message. To see if this happened:

  • From the Chrome extensions page, view the extension console log using: 'Inspect views: background page'. The message is output there
  • Or, look at the 'Status' and 'Outcome' sections below. They should be updated once the test has run.
